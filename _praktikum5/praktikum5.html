1.1 Graafika R-is. ggplot2 ja graafikute grammatika

R-i ks tugevaid klgi on tema jooniste tegemise vimekus. Toome siinkohal esmalt ra paar baaspaketi joonistamise ksku:

# andmed = Massachusettsi osariigi valikuuringu andmed
par(mfrow = c(1, 2), cex = 0.6)  # see rida vimaldab kaks joonist krvuti panna
hist(andmed$AGE, xlab = "Vanus", ylab = "Isikuid", main = "")
plot(andmed$WKHP, andmed$WAGP, xlab = "Ttunde", ylab = "Aastapalk",
     col = (2:3)[factor(andmed$SEX)])
legend("topleft", pch = 19, col = 2:3, legend = levels(factor(andmed$SEX)))

Kuigi R-i baasgraafika on peensusteni konfigureeritav ja sellega saab teha likeerulisi jooniseid, peab tpilisemate andmeid kirjeldavate jooniste saamiseks tegema palju lisatd ja -arvutusi, nagu lalolevastki nha. Niteks barplot(.) ksk soovib argumendiks saada sagedustabelit, mis tuleks siis eelnevalt table(.) ksu abil arvutada.

R-i kasutajate hulgas on viimastel aastatel muutunud populaarseks pakett ggplot2, mis vimaldab lihtsamini joonistada andmestikke kirjeldavaid jooniseid, sealjuures on tulemused visuaalselt sna apetiitsed. Nimelt on ggplot2 arendamisel pandud thele statistiku Edward Tufte soovitusi vrvide valikul ning eriti Leland Wilkinsoni struktureeritud ksitlust andmejoonistest, nn graafikute grammatikat (grammar of graphics).

ldised soovitused (Tufte):

  • tindi/info suhe peab olema vike
  • ei tohiks rhutada mnda elementi, kui see pole teistest olulisem (nt kik vrvid viksid olla sama intensiivsusega)
  • eelistada silmaga lihtsamini hinnatavaid kujundeid (nt tulba krgus tulpdiagrammil vs nurgakraad ringdiagrammil)
  • eemaldada infot mitte kandvad komponendid (nt liba-3D)

Graafikute grammatika (Wilkinson) on kontseptsioon, mille kohaselt graafiku ehitamisel ei tuleks lhtuda mitte graafiku tbist, vaid andmetest. Iga joonis koosneb jrgnevatest osadest:

  • andmed (inimeste palk ja sugu)
  • skaalad (kas esitada palk vrviskaalal vi x-teljel? kas palga esitamiseks sobib nt log-skaala?)
  • statistikud (kas palga puhul kujutada keskmist vi summat)
  • geomeetrilised kujundid (kas keskmine peaks olema mrgitud tulba krgusega vi hoopis punktikesega)
  • koordinaadid (kki sobib polaarkoordinaadistik?)
  • tahud (joonis jagatud erinevateks alamjoonisteks)
  • ldkujundus (font jms)

Paketis ggplot2 on kaks graafikute tekitamise phiksku: ggplot(.) (keerukam) ja qplot(.) (quick plot, lihtsam). Ksk qplot(.) peaks keprasem olema neile, kes on varem kasutanud R-i baasgraafika kske, kuid ldiselt on soovitav ra ppida siiski ggplot(.) kasutamine.

Paketi ggplot2 kskude jaoks on kige phjalikum dokumentatsioon internetis aadressil: http://ggplot2.tidyverse.org/reference/ .

1.2 ggplot2: hajuvusdiagramm; skaalad

Paketi ggplot2 phimte on joonise lesehitamine kihthaaval: esmalt joonise phi ja sellele lisatakse ehk liidetakse (kasutades + mrki) kihtidena erinevad kujundused. Niteks geomeetriliste elementide (punktid, jooned, tulbad jne) lisamiseks/muutmiseks on ksud geom_<elemendi_nimi>.

Alustame hajuvusdiagrammist. Andmestikus maakonnad on info USA 5 osariigi mnede maakondade kohta (425 maakonda). Uurime krgharidusega inimeste osakaalu ja keskmise sissetuleku vahelist seost. Mistlik on seda kujutada hajuvus-diagrammina, kus iga punkt on maakond ning hel teljel on krgharidusega inimeste osakaalu mrkiv tunnus bachelor ja teisel teljel keskmise sissetuleku tunnus per_capita_inc:

mk <- read.table("https://github.com/Rkursus/2021/raw/master/data/maakonnad.txt", sep = " ", header = T)
ggplot(data = mk, mapping = aes(x = bachelor, y = per_capita_inc)) + geom_point()
#                                                    joonise phi  + kujunduselement

Tasub thele panna ksku aes(.) (nagu aesthetics), mida laloleval joonisel funktsiooni argumendile mapping ette anti. Nimelt aes(.) funktsioon aitab siduda graafilisi elemente andmestikus olevate tunnustega.

Lisaks koordinaatidele saab ks punkt veel edasi anda infot niteks vrvi, kuju ja suurusega. Selleks seome funktsiooni aes abil kolm tunnust andmestikust vastavalt vrvi colour, kuju shape ja suuruse size argumentidega:

ggplot(data = mk, mapping = aes(x = bachelor, y = per_capita_inc)) + 
    geom_point(aes(colour = Poverty_factor, shape = State, size = pop_estimate))

Igat tpi tunnuseid ei saa suvaliste joonisehikutega seostada, niteks arvulise tunnusega ei saa siduda punkti kuju (nii palju erineva kujuga punkte pole lihtsalt olemas). Samas aga saab vrviga kujutada nii faktortunnust (nt osariik) kui ka arvulist tunnust (nt krgus merepinnast). Tpsemini on hajuvusdiagrammi hel punktil jrgmised omadused, millega saab infot edasi anda:

  • x (kohustuslik) asukoht x-teljel [num, chr, logical, Factor]
  • y (kohustuslik) asukoht y-teljel [num, chr, logical, Factor]
  • alpha lbipaistvus, viksem vrtus thendab suuremat lbipaistvust [num, chr, logical, Factor]
  • colour vrvus [num, chr, logical, Factor]
  • fill sisemuse vrvus (ainult mne shape vrtuse korral) [num, chr, logical, Factor]
  • shape punkti kuju (kuni 25 erinevat + ise mratavad smbolid) [chr, logical, Factor]
  • size punkti suurus [num]
  • stroke kujundi piirjoone tugevus (ainult mne shape vrtuse korral) [num]

1.3 Tulpdiagramm; elementide asukoht

Graafiku teljed vivad olla seotud ka diskreetse tunnusega (nt Factor), niteks viksime maakondi kujutades siduda x-teljega osariigi:

ggplot(data = mk, aes(x = State)) + geom_bar()

Tekkis tulpdiagramm, kus iga tulp nitab maakondade arvu vastavas osariigis. Ent maakonnad he tulba sees vivad olla erinevad, niteks vaesustaseme (Poverty_factor) poolest. Selleks vib iga tulba vastavate maakondade arvu jrgi ra vrvida. Kuna iga joonise element koosneb piirjoonest ning sisemisest osast, tuleb vastava osa vrvi muutmiseks kasutada kas argumenti colour vi fill:

ggplot(data = mk, aes(x = State)) + geom_bar(aes(fill = Poverty_factor))

Kui on soov esitada maakondade kaupa vaesuse mra osakaalud (mitte absoluutarvud), siis peab kasutama lisaargumenti position = "fill", htlasi on siis mistlik muuta y-telje nimetus:

ggplot(data = mk, aes(x = State)) + 
  geom_bar(aes(fill = Poverty_factor), position = "fill") +
  ylab("osakaal")

Sageli esitatakse tulpdiagrammi tulbad horisontaalselt, selle saavutamiseks tuleb ksk kirja panna sama moodi kui enne, aga lisada tuleb joonise pramine:

ggplot(data = mk, aes(x = State)) + geom_bar() + coord_flip()

Praegu andsime tulpdiagrammi moodustamises ette tunnuse n toorvrtused, vahel on aga tulpdiagramm vaja moodustada valmis sagedustabeli phjal. Selleks, et tulpade krgus mrata mingi tunnuse vrtuste phjal tuleks geom_bar argumendiks lisada stat = "identity"

(sagedustabel <- as.data.frame(table(mk$State)))
ggplot(sagedustabel, aes(x = Var1, y = Freq)) + geom_bar(stat = "identity")

Alternatiivne variant on aga kasutada funktsiooni geom_col()

ggplot(sagedustabel, aes(x = Var1, y = Freq)) + geom_col()

1.4 Veel graafikute tpe

Tulpdiagrammiga sarnane graafik on histogramm, mis sobib arvulise tunnuse jaotuse iseloomustamiseks

ggplot(mk, aes(x = per_capita_inc)) + geom_histogram()
## `stat_bin()` using `bins = 30`. Pick better value with `binwidth`.

Kui uurime tekstiliste vrtustega ja pideva arvutunnuse vahelist seost, siis sobib kasutada karpdiagrammi:

ggplot(data = mk, aes(x = State, y = per_capita_inc)) + geom_boxplot()

Tunnuse jaotuse vrdlemiseks gruppides sobib ka tihedusfunktsiooni hinnagu graafik

ggplot(data = mk, aes(x = per_capita_inc, fill =  State)) + geom_density(alpha = 0.5)

Erinevaid elemente saab joonisel ka korraga kujutada. Niteks aegridade puhul esitada koos mtmispunktid ja neid hendav joon

aeg <- 1:12
tulem <- c(9, 4, 3, 5, 6, 8, 8, 15, 26, 29, 24, 23)
andmed <- data.frame(aeg, tulem )
ggplot(andmed, aes(x = aeg, y = tulem)) + geom_line( ) + geom_point()

Joondiagramm mitmes grupis:

tulem <- c(tulem, c(9:20))
andmed2 <- data.frame(aeg = c(aeg, aeg), tulem, grupp = rep(c("platseebo", "ravim"), each = 12))
ggplot(data = andmed2, aes(x = aeg,y = tulem, colour = grupp))  + 
  geom_line() + geom_point()

Mnikord harvem muidugi vib olla soov esitada pideva ja kategoorilise tunnuse vahelise seose kirjeldamiseks mitte karpdiagramme, aga niteks keskmisi koos usaldusvahemikega. Sellistest keerukamatest vimalustest tuleb hiljem juttu.

Vike kokkuvte joonise tpide valikust:

Tunnus1 Tunnus2 Sobivad joonised geom_<?>
pidev/arvuline - histogramm (tulbad) histogram
pidev/arvuline - tihedus density
kategooriline - tulpdiagramm (tulbad) bar
kategooriline kategooriline tulpdiagramm (tulbad) bar
pidev/arvuline pidev/arvuline hajuvusdiagramm (punktid) point
pidev/arvuline kategooriline karpdiagramm (karbid) boxplot
pidev/arvuline aeg vms jrgnevus joondiagramm (jooned) line

1.5 Joonise jagamine tahkudeks

Sageli on mistlik he suure ja kirju pildi asemel joonistada palju vikeseid sarnase sisuga pilte. Selmet erinevate osariikide maakondade vaesus- ja haridustaseme seost hel ja samal pildil kujutada, viks seda iga osariigi jaoks teha eraldi. Joonise tahkudeks jagamiseks saab kasutada ksku facet_wrap. Sellele argumendile tuleb vrtus anda nn valemi kujul: ridadeks_jagav_muutuja ~ veergudeks_jagav_muutuja. he neist muutujatest vib ka ra jtta:

ggplot(data = mk, aes(x = bachelor, y = perc_poverty)) + geom_point() + 
      facet_wrap(facets = ~State)

Teine sarnane ksk on facet_grid:

ggplot(data = mk, aes(x = bachelor, y = perc_poverty)) +
  geom_point() +
  facet_grid(rows = vars(females_percent > 50), cols = vars(State))

1.6 *Veel joonisele kihtide lisamisest

Paketiga ggplot2 koostatud jooniseid saab salvestada objektina tlauale (st omistada joonis muutujale); valmisolevaid jooniseid saab seetttu lihtsasti muuta ja tiendada. Praktikumi alguses vaatasime juba hajuvusdiagrammi punktidele vrvi lisamist. Teeme selle korra veel lbi, kasutades ka vimalust joonis objektina salvestada

p <- ggplot(data = mk, aes(x = per_capita_inc, y = unemployment_rate)) + 
            geom_point()# tekitatakse joonise objekt, ei kuvata
p # kuvatakse joonis
p + geom_point(mapping = aes(colour = State)) # lisame punktidele vrvi

Kindlasti ei tohi unustada ksku aes(.) (aesthetics), mis aitab siduda graafilisi elemente andmestikus olevate tunnustega. Kui aes(.) funktsiooni ei kasutaks, otsitaks vastavate argumentide vrtuseid mitte andmestikust, vaid tkeskkonnast:

p + geom_point(colour = State) # objekti State otsitakse tkeskkonnast ja ei leita
## Error in layer(data = data, mapping = mapping, stat = stat, geom = GeomPoint, : object 'State' not found

Joonisele saab lisada ka uusi elemente, niteks regressioonikveraid vi teksti. Uute elementide lisamisel vib ette anda ka uue andmestiku argumendiga data:

p + geom_smooth(method = lm) + # lisatakse siluja (praegu lineaarne regressioon)
    geom_text(data = mk[c(34, 48, 65), ], mapping = aes(label = County), size = 3, 
              colour = "red", hjust = 1, vjust = 0)
## `geom_smooth()` using formula 'y ~ x'

Kui joonisele on vaja lisada mingeid ksikuid detaile vi mrkusi, mis ei tulene enam otseselt kasutatud andmestikust, siis saab seda teha ksu annotate(.) abil:

p + 
  annotate("rect", xmin = 2*10^4, xmax = 4*10^4, ymin = 15, ymax = 25, 
           fill = "lightblue", alpha = 0.3, colour = "blue") +
  annotate("segment", x = 3.5*10^4, xend = 3*10^4, y = 12, yend = 20, 
           arrow = arrow(ends = "last", angle = 10)) +
  annotate("text", x = 3.5*10^4, y = 12, label = "vaata\n siia", hjust = 0, vjust = 1)

Eelnevalt oli mainitud, et tunnused, mida tahame joonisel graafiliste elementidega siduda tuleks esitada lbi aes() ksu. Juhul, kui tahame niteks hajuvusdiagrammi punktide vrvi mrata mitte tunnuse phjal muutuvana vaid hoopis fikseerida, siis tuleks vrvi argument just aes(.) ksust vlja jtta, sest vastasel juhul vib tulemus vahel ootamatu olla. Lisaks lisatakse joonisele legend, mis he vrvi fikseerimise korral mtet ei oma. Vrdle jrgmist koodi ja tulemusi:

p + geom_point(color = "blue")
p + geom_point(aes(colour = "blue"))

1.7 Joonise salvestamine

ggplot2-ga tehtud jooniseid saab salvestada ksuga ggsave(.). Kui anda ksule vaid faili nimi (koos laiendiga), siis salvestatakse viimati valmistatud joonis. Kui argumente widht ja height ei kasuta, siis joonise mtmed vetakse joonise akna jrgi.
